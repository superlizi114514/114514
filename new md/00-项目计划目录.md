## 二、登录与账号策略（统一口径）

1. 默认登录方式：邮箱验证码登录 + 图形验证码。
2. 手机号验证定位：登录后可选的安全与账号补全项，用于找回、风控和安全提升。
3. 账号类型并行：邮箱账号与邀请账号并行存在，统一在用户表中标识来源。

## 三、阶段计划

1. 第 1 阶段（基础可用）
2. 范围：邮箱登录、人员与商家点评、榜单展示、基础搜索、基础后台。
3. 输出：最小可用版本（MVP），完成端到端闭环。

1. 第 2 阶段（商业化与运营）
2. 范围：商家认领、推广服务、优惠券、赞助与 VIP、数据看板。
3. 输出：可运营、可变现的版本。

1. 第 3 阶段（体验与规模）
2. 范围：性能优化、内容安全升级、风控完善、统计分析深化。
3. 输出：稳定可扩展版本。

## 四、待办清单（高优先级）

1. 统一登录策略与字段定义，明确邮箱为主、手机号为可选绑定。
2. 落地用户、点评、商家、举报、推广等核心表结构并完成迁移脚本。
3. 建立核心 API 与鉴权中间件，打通前后端。
4. 搭建最小管理后台（审核、举报处理、商家管理）。
