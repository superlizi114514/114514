# 山信黑红榜 — 项目进度与待办

本文档汇总当前已完成与待完成事项，覆盖功能、API 接口与存储实现细节。

## 一、已完成

### 1. 工程与运行基础
1. 前端已搭建：Vue3 + Vite + Vant。
2. 后端已搭建：Express + TypeScript + Prisma。
3. 当前数据库：SQLite（`DATABASE_URL="file:./dev.db"`）。
4. 后端开发脚本使用 `tsx`，避免 Node 24 + ts-node/esm 崩溃。

### 2. 登录与账号
1. 邮箱验证码登录 + 图形验证码。
2. 手机号绑定为登录后可选项。
3. 已落地 API：
1. `GET /api/auth/captcha`
2. `POST /api/auth/send-email-code`
3. `POST /api/auth/login`
4. `POST /api/auth/send-phone-code`
5. `POST /api/auth/bind-phone`
4. 存储实现：
1. `CaptchaCode`、`EmailCode`、`SmsCode` 落库。
2. `User` 通过邮箱登录自动 `upsert`。

### 3. 人员点评核心功能
1. 人员添加与列表。
2. 人员点评提交与列表。
3. 点评规则：内容 10-200 字，类型 red/black。
4. 每日同一人点评次数：普通 3 次，VIP 5 次。
5. 人员榜单（按点评数）。
6. 已落地 API：
1. `GET /api/profiles`
2. `POST /api/profiles`
3. `GET /api/profiles/:id/reviews`
4. `POST /api/profiles/:id/reviews`
5. `GET /api/rankings/profile`
7. 存储实现：`Profile`、`ProfileReview` 已落库并关联 `User`。

### 4. 商家模块（第一版）
1. 商家列表、添加、点评、榜单前端已可用。
2. 已落地 API：
1. `GET /api/merchants`
2. `POST /api/merchants`
3. `GET /api/merchants/:id/reviews`
4. `POST /api/merchants/:id/reviews`
5. `GET /api/rankings/merchant`
3. 存储实现：`Merchant`、`MerchantReview` 已落库。

### 5. 搜索与排序
1. 搜索支持人员与商家。
2. 支持红/黑筛选。
3. 支持最新/最热排序。
4. 最热采用红黑权重：红=2，黑=1。
5. 已落地 API：`GET /api/search?keyword=&type=&sort=`

### 6. 管理后台（最小版）
1. 用户可举报人员点评。
2. 管理员可查看待处理举报、处理记录。
3. 支持分页与筛选（时间、关键字、状态）。
4. 已落地 API：
1. `POST /api/reports/profile`
2. `GET /api/admin/reports?page=&pageSize=&keyword=&start=&end=`
3. `GET /api/admin/reports/history?page=&pageSize=&status=&keyword=&start=&end=`
4. `POST /api/admin/reports/:id/approve`
5. `POST /api/admin/reports/:id/reject`
5. 管理员鉴权使用 `x-admin-token`（`ADMIN_TOKEN`）。
